# voice-notes-bot

Личный Telegram-бот — голосовой ассистент для дневных заметок. Надиктовываешь
голосовые, бот расшифровывает их (Whisper), структурирует (Claude) и даёт выжимку.
В конце дня `/day` собирает сводку — её кладёшь в Cowork «daily dollar balance».

## Как пользоваться

- **голосовое или текст** → расшифровка + структура + короткая выжимка, заметка
  сохраняется в текущий день.
- **правка любой записи** голосом/текстом по номеру из `/list`: «удали вторую запись»,
  «исправь запись 3: сумма 500», «переделай предпоследнюю». Можно ссылаться и по смыслу:
  «исправь запись про аренду».
- **/list** → показать записи за сегодня (с номерами, временем и выжимкой).
- **/undo** → удалить последнюю запись (бот покажет, что удалил).
- **/day** → сводка за сегодня (или `/day ГГГГ-ММ-ДД` за прошлый день).
- **/week** → сводка за последние 7 дней.
- **/history** → список дней, за которые есть записи.
- **/find текст** → поиск по всем записям за всё время.
- **/clear** → очистить заметки за сегодня.
- **/start** → приветствие и твой user id.

Внизу чата — **кнопки** для основных действий (📋 Список, 🗓 Сводка дня,
↩️ Удалить последнюю, 🧹 Очистить день). Плюс список команд в меню бота (кнопка «Menu»).

Правки распознаёт лёгкий классификатор; при сомнении сообщение считается обычной
заметкой (чтобы случайно не удалить/переписать). При удалении бот выводит удалённый
текст — если что, можно продиктовать заново.

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env   # заполни значения
python main.py         # long-polling, публичный URL не нужен
```

Нужны:
- токен бота от [@BotFather](https://t.me/BotFather);
- ключ Anthropic (структурирование и сводка);
- ключ OpenAI (Whisper для голосовых);
- свой `user id` — узнаёшь, отправив боту `/start`.

Для сервера подойдёт `Procfile` (`worker: python main.py`) на Railway/Render/Fly.

## Хранение

Заметки лежат по файлу на день: `<NOTES_DIR>/<user_id>/<YYYY-MM-DD>.json`.
Старые дни не удаляются — история копится за всё время. Папка `notes/` в `.gitignore`.

Все даты и метки времени — **по Москве** (UTC+3). Это важно для границы суток: заметка,
надиктованная ночью по МСК, попадает в правильный день, а не во вчерашний по UTC.

**Постоянное хранение на Railway:** файловая система контейнера эфемерна и сбрасывается
при каждом деплое. Чтобы история переживала деплои, подключи **Volume** и укажи путь
монтирования в переменной `NOTES_DIR` (например, Volume на `/data`, тогда
`NOTES_DIR=/data/notes`).

## Безопасность

- Доступ **fail-closed**: без `ALLOWED_USER_ID` бот не пускает никого.
- `.env` и `notes/` — в `.gitignore`.
